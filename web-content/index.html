Práctica: Alta Disponibilidad con Balanceo de Carga

Módulo ASIX - Proyecto desarrollado por Lautaro.

Arquitectura del Sistema

En este proyecto Nginx distribuye las peticiones entrantes entre dos servidores Apache internos compartiendo un mismo volumen de datos.

Configuración Nginx Activa

Este es el archivo default.conf que gobierna el balanceo de carga en nuestro contenedor de Nginx:

upstream apache_backends {
    # El bloque 'upstream' define al grupo de servidores backend.
    # Usando Round Robin (por defecto), Nginx asigna las peticiones 1 a 1.
    server web_lautaro_1:80;
    server web_lautaro_2:80;
}

server {
    # Escucha en el puerto 80 del contenedor Nginx
    listen 80;
    server_name localhost;

    # Se pasan las cabeceras originales del cliente hacia los Apache
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;

    location / {
        # Se envía la petición equilibrada al grupo 'apache_backends'
        proxy_pass http://apache_backends;

        # Se desactiva la memoria caché para que el proxy siempre 
        # envíe la solicitud nueva y demuestre el cambio de servidores
        proxy_no_cache 1;
        proxy_cache_bypass 1;
        
        # 'X-Upstream' devuelve al Javascript qué IP interna exacta nos atendió
        add_header X-Upstream $upstream_addr always;
        add_header X-Balancer "Alta_Disponibilidad_Lautaro" always;
    }
}
